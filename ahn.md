# 학습하는 방식
방금 설명해 드린 **`app.py`의 데이터 수집 및 모드 변경 로직**을 핵심만 깔끔하게 정리해 드릴게요.

---

### **1. 키보드 역할 총정리**

이 프로그램은 **키보드 입력**을 통해 **"어떤 데이터를(Mode)"**, **"어떤 이름으로(Label)"** 저장할지 결정합니다.

| 구분 | 키(Key) | 기능 | 설명 |
| :--- | :---: | :--- | :--- |
| **모드 변경** | **`k`** | **정적 모드 (Mode 1)** | 손 **모양**(Keypoint) 데이터를 수집할 준비를 합니다. (`keypoint.csv` 사용) |
| | **`h`** | **동적 모드 (Mode 2)** | 손 **움직임**(History) 데이터를 수집할 준비를 합니다. (`point_history.csv` 사용) |
| | **`n`** | **일반 모드 (Mode 0)** | 데이터 수집을 멈추고 관전 모드로 돌아갑니다. (기본 상태) |
| **저장 실행** | **`0` ~ `9`** | **라벨 지정 및 저장** | 현재 손동작을 해당 숫자(ID)의 정답으로 **즉시 저장**합니다. |

---

### **2. 데이터 수집 흐름 (Workflow)**

실제로 데이터를 모을 때는 **[모드 선택] $\rightarrow$ [동작 취하기] $\rightarrow$ [저장]** 순서로 진행됩니다.

1.  **준비:** 키보드 **`k`**(모양) 또는 **`h`**(움직임)를 눌러 수집 모드를 켭니다.
2.  **행동:** 카메라 앞에서 원하는 손동작(예: 주먹 쥐기, 왼쪽으로 손 흔들기 등)을 취합니다.
3.  **기록:** 동작을 유지한 채로 키보드 숫자키(예: **`0`**)를 누릅니다.
    * *찰칵!* 📸 $\rightarrow$ 그 순간의 좌표 데이터가 `0`번 라벨로 CSV 파일에 한 줄 추가됩니다.

### **3. 코드 내부 동작 원리**

* **`select_mode(key, mode)`:**
    * `k`, `h`, `n`을 감지하여 전역 변수 `mode`의 상태(0, 1, 2)를 바꿉니다.
    * `0`~`9`를 감지하여 `number` 변수에 저장할 라벨 번호를 담습니다.
* **`logging_csv(number, mode, ...)`:**
    * `mode`가 1이나 2일 때만 작동합니다.
    * `number`가 들어오는 순간, 현재의 손 좌표 리스트를 CSV 파일에 `append`(이어쓰기) 합니다.

이제 이 원리를 알았으니, 웹캠을 켜고 `k`를 누른 뒤 주먹을 쥐고 `0`번을 연타하면, 나만의 '주먹 데이터셋'을 만들 수 있게 된 것입니다!
